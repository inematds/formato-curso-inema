Quatro receitas prontas que tiram trabalho chato da sua frente — cada uma é um hook curto que você cola e esquece.

↓ role para estudar

01 Formatar ao salvar

O clássico. Depois de o Claude editar um arquivo, rode o formatador nele. É PostToolUse com matcher Edit|Write:

{ "hooks": { "PostToolUse": [ {
  "matcher": "Edit|Write",
  "hooks": [ { "type": "command",
    "command": "f=$(jq -r .tool_input.file_path); prettier --write \"$f\"" } ]
} ] } }

O caminho do arquivo vem do stdin. Troque prettier por black, gofmt, o que for do seu projeto. Código sempre limpo, sem você lembrar.

02 Avisar quando terminar

Você saiu pra pegar um café e o Claude terminou faz tempo. O evento Stop resolve: dispara um aviso quando ele acaba de responder.

# Stop → notificação do sistema
notify-send "Claude" "terminou 👋"   # linux
# osascript -e 'display notification "terminou"'  (mac)

Um afplay/paplay com um som curto também funciona. Pequeno, mas muda seu fluxo: você para de vigiar o terminal.

03 Registrar os comandos

Quer um histórico do que foi rodado? PreToolUse no Bash grava cada comando num arquivo antes de ele executar:

# PreToolUse, matcher "Bash" → append num log
jq -r '.tool_input.command' >> "$CLAUDE_PROJECT_DIR/.claude/cmd.log"
exit 0

Sai com exit 0 (não barra nada, só observa). Depois você tem um diário do que a sessão fez — ótimo pra auditar ou desfazer.

04 Injetar contexto no começo

O evento SessionStart roda quando a sessão abre, e o que ele imprime vira contexto pro Claude. Use pra ele já começar sabendo do projeto:

# SessionStart → estado atual entra no contexto
echo "Branch: $(git branch --show-current)"
echo "Últimos commits:"; git log --oneline -5

Assim o Claude abre a sessão já ciente do branch, do que mudou, das regras do projeto — sem você colar nada à mão.
