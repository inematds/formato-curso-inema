Hooks que
protegem

Transformar "por favor não faça isso" numa trava de verdade: barrar o perigoso antes de acontecer, ou pedir confirmação.

↓ role para estudar

01 A ideia de trava

Uma trava é um hook de PreToolUse: ele roda antes da ferramenta e decide se ela pode seguir. O sinal é o código de saída:

  • exit 0 — tudo bem, pode rodar.
  • exit 2 — barra. O Claude não executa, e o que você imprimiu no stderr vira o motivo.

Repare: a trava não "avisa depois" — ela impede antes. É a diferença entre um extintor e um relatório de incêndio.

02 Ler o que vai rodar

Pra decidir, o hook precisa ver o comando. Ele vem no JSON do stdin; você pega com jq e testa:

# .claude/hooks/guarda.sh (PreToolUse, matcher "Bash")
cmd=$(jq -r '.tool_input.command')
if echo "$cmd" | grep -qE 'rm -rf|:(){:'; then
  echo "bloqueado: comando destrutivo" >&2
  exit 2
fi
exit 0

A régua é sua: um grep por padrões perigosos já pega a maioria dos acidentes. Comece simples e vá endurecendo conforme aprende o que te morde.

03 Casos que valem a pena

Três travas que salvam o dia mais cedo do que tarde:

  • Nada de rm -rf em caminhos amplos — o clássico acidente irreversível.
  • Não tocar em segredos — barrar edição de .env, *.pem, chaves.
  • Não empurrar na main — barrar git push direto no branch protegido.

Não precisa prever tudo. Cada susto vira uma linha nova na trava — e ela fica mais esperta com o tempo.

04 Pedir confirmação, não só barrar

Nem tudo é preto no branco. Às vezes você não quer proibir, quer ser avisado pra confirmar. Em vez do exit 2, o hook pode devolver um JSON pedindo confirmação:

{ "hookSpecificOutput": {
    "hookEventName": "PreToolUse",
    "permissionDecision": "ask",
    "permissionDecisionReason": "confirmar antes de deploy" } }

Assim o Claude pausa e pergunta. Três posturas: allow (libera), deny (barra, como o exit 2) e ask (confirma). Use ask quando o certo depende do contexto.

Indo mais fundo

O exit 2 é o caminho rápido; a saída JSON é a versão fina, com motivo estruturado. Comece pelo exit code — migre pro JSON quando precisar de "ask" ou mensagens mais ricas.
